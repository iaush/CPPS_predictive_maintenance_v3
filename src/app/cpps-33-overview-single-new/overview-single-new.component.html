Prescriptive maintenance aims to provide a sequence of prescriptive actions to best optimise maintenace cost when a machine failure is detected

How It Works

Description

Upon detecting performance degradation, algorithm 1 attempts to determine the root cause of the issue. This helps to narrow down an entire list of possible maintenace action to only those related to the determined root cause. Algorithm 2 thus focuses on the forementioned narrowed down list of maintenace actions to generate a sequence of recommended actions to optimise the maintenance cost.


Problem

  1. Current Predictive maintenance methodologys predict failure but do not provide information on what actions to take

  2. There are many possible maintenance actions for a faulty machine component problem but it is often hard to detemine which action is the most appropraite in terms of cost and effectivenss

Deep reinforcement learning

1. What is the difference between reinforcement learning and deep reinforcement learning?

The difference between them is that deep learning is learning from a training set and then applying that learning to a new data set, while reinforcement learning is dynamically learning by adjusting actions based in continuous feedback to maximize a reward.

2. What is the advantage of our algorithm?

  1. Traditional RL algorithms wherein an agent learns from reward, our algorithm learns a mapping from desired rewards and observations to actions.
  2. Our algorithm includes two parallel algorithms:
    • A1 performs execution and exploration
    • A2 performs supervised learning. A1 generates actions based on known maximum possible cumulative rewards using the current version of the model trained by A2.
  3. For the sake of exploration, random actions are taken occasionally. The observations, actions and rewards are saved in a buffer. A2 uses historical data that is extracted from the buffer to train the model that maps the observation and maximum possible cumulative rewards to optimal action.
  4. Relying purely on offline data makes offline RL a perfect choice for maintenance decision-making wherein storing historical data to learn a policy is much more cost-effective than developing a simulator that can model the problem accurately.
